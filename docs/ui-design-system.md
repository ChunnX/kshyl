# UI 设计系统

## 设计方向

产品面向 70+ 长辈和协助校对的家人，界面采用“安心、清晰、少打扰”的移动工具风格。老人端优先语音动作，家人端优先阅读和编辑。

## 视觉原则

- 大字号：标题约 52rpx，正文 30-34rpx，主按钮 36rpx。
- 大触控：主按钮高度不低于 116rpx。
- 少选择：每屏保留一个最重要动作，其余动作降级。
- 高对比：深蓝文字、白色面板、蓝绿按钮，避免低对比灰字。
- 明确状态：录音中、准备开始、待校对、已校对都要可见。
- 配图克制：插画只帮助识别页面主题，不抢操作焦点。

## 色彩

- 主色：`#0369A1`，用于主要按钮和关键状态。
- 辅色：`#22C55E`，用于录音、授权、完成等正向状态。
- 背景：`#F0F9FF`，柔和浅蓝。
- 正文：`#0C4A6E` / `#082F49`。
- 提示：`#ECFDF5` / `#DCFCE7`。
- 强调：`#FDE68A`，仅用于插画点缀。

## 页面约定

- 首页：展示“每天 5 分钟”的低压力入口，主按钮为开始讲故事。
- 语音陪聊页：模式选择只保留“对话模式 / 倾诉模式”，避免复杂设置；底部始终保留录音主按钮。
- 录音页：用大圆形状态区承载“准备好了 / 正在听”，避免复杂录音控件。
- 故事页：卡片展示故事，状态标签区分待校对和已校对。
- 校对页：编辑器大字号、高边框，提示家人保留长辈语气。
- 成书页：用书本插画降低抽象感，结果区只显示当前书稿状态。
- 纪念对话页：授权提示必须前置，避免用户误以为可以直接模拟真人。
